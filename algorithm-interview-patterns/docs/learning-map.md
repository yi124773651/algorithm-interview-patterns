## 一、总学习思路

很多人刷题卡住，不是因为不会写代码，而是因为：

- 看不出题目的结构
- 不知道应该优先试哪种方法
- 模板会写，但不理解为什么有效

所以训练顺序不应该是：

- 题海战术
- 背题解
- 反复记答案

而应该是：

1. **先识别结构**
2. **再理解方法为什么成立**
3. **最后稳定输出模板**

你可以把刷题理解成一句话：

> 不断训练“题面语言”到“结构模型”的翻译能力。

## 1. 滑动窗口

**代码目录：** `src/main/java/no1_slidingwindow`

### 心理表征

一个“可伸缩的连续区间”。

右边界负责扩张，左边界负责收缩。  
窗口内部始终维护某个条件。

### 适合处理的问题

- 连续子数组 / 子串
- 最长满足条件
- 最短满足条件
- 包含 / 覆盖 / 去重

### 基本步骤

1. 明确窗口表示什么
2. 明确窗口内部维护哪些量
3. 右指针扩张
4. 不合法时左指针收缩
5. 更新答案

### 识别信号

- 题目强调“连续”
- 想找一个满足条件的区间
- 条件能随着边界移动动态维护

### 代表题

- 无重复字符的最长子串
- 最小覆盖子串
- 长度最小的子数组
- 找到字符串中所有字母异位词

### 练习目标

- 会区分“固定窗口”和“可变窗口”
- 会区分“至少 / 至多 / 恰好”三种约束

## 2. 双指针

**代码目录：** `src/main/java/no2_twopointertechnique`

### 心理表征

两个位置在序列中协同移动。  
它们像两只手，通过相对位置关系减少枚举。

### 适合处理的问题

- 有序数组配对
- 原地删除 / 去重
- 链表快慢问题
- 首尾夹逼

### 基本步骤

1. 明确两个指针职责
2. 根据比较关系移动指针
3. 利用有序性或相对关系剪掉无效搜索

### 识别信号

- 有序数组
- 两数和 / 三数和
- 链表中点 / 环
- 原地操作数组

### 代表题

- 两数之和 II
- 三数之和
- 盛最多水的容器
- 删除有序数组中的重复项
- 环形链表

### 练习目标

- 能区分左右夹逼、同向双指针、快慢指针
- 能说清楚为什么某个指针移动后不会漏解

## 3. BFS

**代码目录：** `src/main/java/no3_bfs`

### 心理表征

按层扩散的搜索前沿。  
每一层通常代表距离、步数或状态推进一轮。

### 适合处理的问题

- 无权图最短路径
- 最少步数 / 最少操作
- 多源扩散
- 层序遍历

### 基本步骤

1. 建立节点与边
2. 起点入队
3. visited 防重复
4. 按层遍历
5. 到达终点即返回

### 识别信号

- 最短步数
- 最少转换次数
- 从一个或多个起点扩散
- 每步代价相同

### 代表题

- 二叉树层序遍历
- 腐烂的橘子
- 单词接龙
- 01 矩阵

### 练习目标

- 会写单源 BFS、多源 BFS
- 会理解“队列中的顺序就是距离顺序”

## 4. DFS / 回溯

**代码目录：** `src/main/java/no4_dfsbacktracking`

### 心理表征

一棵“做选择 -> 深入 -> 撤销 -> 换分支”的决策树。

### 适合处理的问题

- 枚举全部方案
- 路径搜索
- 组合、排列、子集
- 棋盘、切割、装填类问题

### 基本步骤

1. 定义递归状态
2. 定义终止条件
3. 枚举当前可选项
4. 做选择
5. 递归
6. 撤销选择

### 识别信号

- 所有方案
- 所有路径
- 是否存在某种可行方案
- 组合 / 排列 / 切分 / 分配

### 代表题

- 子集
- 全排列
- 组合总和
- N 皇后
- 单词搜索
- 分割回文串

### 练习目标

- 理解树层去重、树枝去重
- 理解剪枝不是玄学，而是约束提前判断

---

## 5. 动态规划

**代码目录：** `src/main/java/no5_dynamicprogramming`

### 心理表征

一张状态账本。  
每个状态都表示某个子问题的答案。

### 适合处理的问题

- 最优值
- 可行性
- 方案数
- 暴力递归会重复计算的题

### 基本步骤

1. 定义 `dp` 含义
2. 找状态转移
3. 定义初始值
4. 安排遍历顺序
5. 看能否压缩空间

### 识别信号

- 最长、最多、最少
- 方案数、可不可行
- 当前结果依赖更小规模结果
- 递归写出来有很多重复子问题

### 代表题

- 爬楼梯
- 打家劫舍
- 最长递增子序列
- 零钱兑换
- 编辑距离
- 不同路径
- 目标和

### 练习目标

- 先会说状态，再写代码
- 学会从“最后一步发生了什么”倒推转移

## 6. 贪心

**代码目录：** `src/main/java/no6_greedy`

### 心理表征

一个“局部最优且不后悔”的选择规则。

### 适合处理的问题

- 区间调度
- 覆盖问题
- 资源分配
- 跳跃类问题
- 排序后扫描问题

### 基本步骤

1. 找局部最优规则
2. 证明不会吃亏
3. 排序或扫描实现

### 识别信号

- 每一步都像在选一个当前最优动作
- 选完后通常不需要回头
- 经常能通过排序建立顺序

### 代表题

- 分发饼干
- 跳跃游戏
- 跳跃游戏 II
- 无重叠区间
- 划分字母区间
- 加油站

### 练习目标

- 不只会写，还要能解释“为什么这样贪不会错”
- 能区分“贪心可做”与“其实需要 DP”

## 7. 二分搜索

**代码目录：** `src/main/java/no7_binarysearch`

### 心理表征

不是找数，而是在**单调答案空间**里找边界。

### 适合处理的问题

- 有序数组查找
- 第一个满足条件的位置
- 最小可行值 / 最大可行值
- 答案二分

### 基本步骤

1. 确定搜索区间
2. 写 `check(mid)`
3. 根据真假缩区间
4. 收敛到边界

### 识别信号

- 最小的 x 使得……
- 最大的 x 仍然……
- 存在明显单调性

### 代表题

- 搜索插入位置
- 在排序数组中查找元素的第一个和最后一个位置
- Koko 吃香蕉
- 爱吃香蕉的珂珂
- 在 D 天内送达包裹的能力

### 练习目标

- 学会从“求答案”转成“判定可行性”
- 熟悉左边界、右边界写法

## 8. 栈 / 单调栈

**代码目录：** `src/main/java/no8_stack`

### 心理表征

栈是后进先出的延迟处理结构。  
单调栈是在此基础上加入“有序约束”。

### 适合处理的问题

- 括号匹配
- 表达式计算
- 最近更大 / 更小元素
- 柱状图面积类

### 基本步骤

1. 确定栈里存什么
2. 新元素来时弹出失效候选
3. 保持单调性
4. 弹出时或遍历后结算答案

### 识别信号

- 最近更大 / 更小
- 左边第一个 / 右边第一个
- 温度、跨度、矩形、面积

### 代表题

- 有效括号
- 每日温度
- 下一个更大元素
- 柱状图中最大的矩形
- 接雨水（也可双指针）

### 练习目标

- 会区分单调递增栈和单调递减栈
- 会理解“为什么被弹出的元素以后永远没用了”

## 9. 前缀和

**代码目录：** `src/main/java/no9_prefixsum`

### 心理表征

历史累计快照。  
任何区间都能变成两个历史量之差。

### 适合处理的问题

- 区间和
- 子数组和统计
- 二维区域和
- 累计出现次数问题

### 基本步骤

1. 构建前缀和
2. 用前缀差值表示区间
3. 若统计个数，配合哈希表记录历史前缀频次

### 识别信号

- 连续区间和
- 子数组和为 k
- 多次查询区间和
- 条件可转成“前缀差值满足某关系”

### 代表题

- 和为 K 的子数组
- 区域和检索
- 连续数组
- 二维区域和检索

### 练习目标

- 熟悉一维和二维前缀和
- 理解“前缀和 + 哈希表”的组合威力

## 10. 并查集

**代码目录：** `src/main/java/no10_unionfind`

### 心理表征

一片动态合并的森林。  
每棵树代表一个连通块。

### 适合处理的问题

- 判断连通
- 合并集合
- 动态加边
- 连通分量管理

### 基本步骤

1. 初始化父节点
2. `find` 找根
3. `union` 合并
4. 路径压缩和按秩合并优化

### 识别信号

- 是否属于同一组
- 是否连通
- 多次合并后反复查询
- 动态连边

### 代表题

- 冗余连接
- 岛屿数量（并查集写法）
- 账户合并
- 等式方程的可满足性

### 练习目标

- 会手写路径压缩
- 理解并查集适合“动态维护”，而不是“每次现搜”

## 11. 拓扑排序

**代码目录：** `src/main/java/no11_topologicalsort`

### 心理表征

一个“依赖逐步解锁”的过程。  
当前没有前置依赖的点，先做。

### 适合处理的问题

- 课程安排
- 任务调度
- 依赖顺序
- DAG 分层推进

### 基本步骤

1. 建图
2. 统计入度
3. 入度为 0 的点入队
4. 弹出后更新后继入度
5. 看是否处理完全部节点

### 识别信号

- 必须先 A 再 B
- 课程先修关系
- 判断是否存在环
- DAG 上按阶段推进

### 代表题

- 课程表
- 课程表 II
- 外星文字典
- 矩阵中的最长递增路径（DAG 建模）

### 练习目标

- 会用入度数组写标准拓扑排序
- 理解“拓扑排序 = 特殊语义下的 BFS 分层”

## 三、模式之间的边界与联系

### 滑动窗口 vs 前缀和

- 都处理连续区间
- 滑动窗口强调“动态维护合法区间”
- 前缀和强调“快速计算区间累计量”

### 双指针 vs 滑动窗口

- 滑动窗口通常是同向双指针的一种特例
- 但双指针更广，包含左右夹逼、快慢指针等模式

### DFS / 回溯 vs 动态规划

- 回溯在遍历决策树
- DP 在压缩重复子问题
- 记忆化搜索是二者的桥梁

### BFS vs DFS

- BFS 更适合最短步数
- DFS 更适合遍历、存在性、枚举方案

### 贪心 vs 动态规划

- 贪心只保留一个当前最优选择
- DP 保留多个状态分支
- 是否能贪，关键在于能否证明“不后悔”

### 并查集 vs DFS / BFS

- DFS / BFS 适合一次性遍历图
- 并查集适合边不断加入、连通性反复查询的场景

### 拓扑排序 vs BFS

- 拓扑排序很多时候就是一种 BFS
- 区别在于“层”的含义从距离变成依赖阶段

## 四、推荐刷题路径

### 第 1 轮：建立结构感

顺序建议：

1. 滑动窗口
2. 双指针
3. BFS
4. DFS / 回溯
5. 前缀和

目标：

- 看出题目大致属于区间、图还是决策树

### 第 2 轮：建立抽象能力

顺序建议：

1. 动态规划
2. 贪心
3. 二分搜索
4. 栈 / 单调栈

目标：

- 能处理更抽象的状态、单调性、局部最优证明

### 第 3 轮：建立图结构建模能力

顺序建议：

1. 并查集
2. 拓扑排序

目标：

- 能把“题面关系”翻译成图结构
- 能理解连通、依赖、无环、分层推进等概念

## 五、复盘模板

每做完一道题，建议用下面 6 个问题复盘：

1. **这题维护的核心结构是什么？**
   - 区间、状态、图、决策树、单调栈、连通块？

2. **暴力方法是什么？瓶颈在哪？**
   - 枚举太多？
   - 重复子问题？
   - 区间求值太慢？
   - 搜索空间过大？

3. **我为什么想到这个方法？**
   - 连续区间？
   - 最短步数？
   - 单调性？
   - 最近更大 / 更小？
   - 动态连通？

4. **核心不变量是什么？**
   - 窗口始终合法？
   - 队列按层？
   - `dp[i]` 含义稳定？
   - 栈保持单调？
   - 并查集根节点代表集合？

5. **这题最容易写错的点是什么？**
   - 边界
   - 初始化
   - 去重
   - 剪枝
   - 下标
   - 判定函数

6. **如果题目换个壳，我还能认出来吗？**
   - 数组换字符串
   - 字符串换矩阵
   - 树换图
   - 题面从叙述变依赖关系

## 最后一句

训练算法模式的目标，不是“背出更多题”，而是：

> 逐步建立一套稳定的结构识别系统。

当你看到题目时，不再先想“我做过没有”，  
而是先想：

- 这题是连续区间吗？
- 这是最短路吗？
- 这是决策树吗？
- 这是重复子问题吗？
- 这是单调答案空间吗？
- 这是连通性或依赖关系吗？

当这套识别系统稳定下来，刷题速度和迁移能力都会明显提高。
